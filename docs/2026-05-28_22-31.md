# 修改记录: 2026-05-28 22:31

## 诊断

此前 17 次训练均从 step 3000 的 checkpoint resume（最早来自 May27_00-56 的首次训练）。首次训练 step 0→3000 达到 mean_reward +36.63，但后续每次 resume 都持续退化，最后一次 (May28_02-33) 崩到 mean_reward -13.47、smoothness -6.27。

根因推测: PPO 从旧 checkpoint resume 时，optimizer 状态（Adam 动量）与当前 reward 结构不匹配，导致更新方向错误。旧 checkpoint 经历过多轮 reward weight 修改，其价值函数估计与当前 reward 严重不一致。
